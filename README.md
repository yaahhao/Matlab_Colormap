# ColorKit MATLAB Colormap Toolkit

[English](README.md) | [中文](README_CN.md)

Author: zhaoyh  
Email: zhao2025@mail.sustech.edu.cn

ColorKit is a compact MATLAB toolkit for building, browsing, editing, and reusing colormaps in research figures. It includes about 80-100 built-in named palettes, image-based color picking, screenshot colorbar recovery, dominant-color extraction, palette-card browsing, user presets, and an interactive `ColormapDesigner` GUI.

## What You Can Do

- Browse palettes as visual cards before choosing one for a figure.
- Call a palette by name or catalog ID and apply it to `imagesc`, `surf`, `contourf`, line plots, scatter plots, or classed heatmaps.
- Pick colors from any imported image by clicking pixels, or pass fixed pixel coordinates for reproducible scripts.
- Recover a colormap from a screenshot colorbar by sampling the first and last points.
- Extract image theme colors with either grid quantization or clustering.
- Save any `N x 3` RGB matrix as a user preset.
- Add personal `RecommendedUse` notes as your own plotting habits develop.
- Open `ColormapDesigner` for manual editing, previewing, tuning, applying, and exporting colormaps.

## Quick Start

Clone or download this repository, open MATLAB, and make the project folder your current folder. Then add the toolbox to the MATLAB path:

```matlab
addpath(genpath(pwd))
```

Call a named 256-color colormap:

```matlab
C = ColorKit('palette', 'Blue', 'nColors', 256);
imagesc(peaks(200));
colormap(C);
colorbar;
title('Blue');
```

Use the lightweight wrapper if you only need the colormap:

```matlab
C = getPresetColormap('Viridis', 256);
```

You can also call palettes by catalog ID. IDs are shown in `ColorKit('catalog')` and on every color card:

```matlab
catalogTable = ColorKit('catalog');
C = ColorKit('palette', 13, 'nColors', 256);
C = getPresetColormap(13, 256);
```

Open the GUI editor:

```matlab
ColormapDesigner
```

Run the complete demo:

```matlab
Demo_ColorKitFull
```

## Core Calling Pattern

Most calls follow this pattern:

```matlab
output = ColorKit(action, mainInput, options...)
```

Examples:

```matlab
C = ColorKit('palette', 'Blue', 'nColors', 256);
C = ColorKit('palette', 13, 'nColors', 256);
C = ColorKit('ramp', 'BlueWhiteRed', 'nColors', 256);
C = ColorKit('pick', img, 'preview', true);
C = ColorKit('imagebar', img, 'samples', 16, 'nColors', 256);
C = ColorKit('theme-grid', 8, img);
row = ColorKit('add-preset', C, 'presetName', 'My_BlueOrange');
row = ColorKit('set-use', 'My_BlueOrange', 'phase display');
```

Continuous colormaps return an `N x 3` double matrix in `[0, 1]`. Qualitative palettes also return an RGB matrix; when you request more colors than the native anchors, the toolbox expands them while avoiding a completely flat repeated color.

## Main Actions

| Action | Example | Output | Meaning |
|---|---|---|---|
| `palette` | `ColorKit('palette','Blue','nColors',256)` or `ColorKit('palette',13,'nColors',256)` | `N x 3` RGB | Read a palette by name or catalog ID. |
| `ramp` | `ColorKit('ramp','BlueWhiteRed','nColors',256)` | `N x 3` RGB | Readable alias for `palette` with a requested length. |
| `cards` | `ColorKit('cards','blue',1,40,'nColors',256)` | figure handle | Browse palettes as color cards. |
| `search` | `ColorKit('search','diverging')` | table | Search name, palette group, type, use note, source style, and custom group names. |
| `catalog` | `ColorKit('catalog')` | table | Return the full palette metadata table. |
| `about` | `ColorKit('about')` | table | Summarize palette-group counts. |
| `pick` | `ColorKit('pick', img, 'preview', true)` | `M x 3` or `N x 3` RGB | Pick image colors from clicked or fixed pixels. |
| `imagebar` | `ColorKit('imagebar',img,'samples',16,'nColors',256)` | `N x 3` RGB | Sample a colorbar line from an image and densify it. |
| `theme-grid` | `ColorKit('theme-grid',8,img)` | `k x 3` RGB | Extract dominant image colors by quantized color statistics. |
| `theme-cluster` | `ColorKit('theme-cluster',8,img)` | `k x 3` RGB | Extract dominant image colors by k-means or internal fallback clustering. |
| `add-preset` | `ColorKit('add-preset', C, 'presetName','My_BlueOrange')` | table row | Save an existing `N x 3` RGB matrix as a user preset. |
| `set-use` | `ColorKit('set-use','My_BlueOrange','phase display')` | table row | Update your personal recommended-use note. |

## Common Parameters

| Parameter | Used By | Meaning |
|---|---|---|
| `'nColors', N` | `palette`, `ramp`, `pick`, `imagebar`, `cards` | Requested output color count. Use 256 for continuous maps, 5-12 for categories, and 512 or 1024 for very smooth image export. |
| `'preview', true/false` | palette and image workflows | Show a preview figure. For scripted batch runs, use `false`. |
| `'points', [x y; ...]` | `pick` | Fixed image pixel coordinates for reproducible image color picking. |
| `'endpoints', [x1 y1; x2 y2]` | `imagebar` | First and last points of the colorbar line in image pixel coordinates. |
| `'samples', N` | `imagebar` | Number of equally spaced anchor samples between the two colorbar endpoints. |
| `'columns', n` | `cards` | Number of columns in the card browser. Internally constrained to a practical range. |
| `'visible', true/false` | `cards` | Show or hide the card figure. Useful when exporting cards without displaying them. |
| `'saveAs', file` | `cards` | Export the card browser figure through `exportgraphics`. |
| `'presetName', name` | `add-preset` | Name for a new user preset. Invalid characters are normalized. |
| `'paletteGroup'` | `add-preset` | Preferred user palette group. Use an existing group or your own group name; default `"none"` maps to `"Custom"`. |
| `'type'` | `add-preset` | Optional palette type: `sequential`, `diverging`, `cyclic`, `qualitative`, or `"none"`. |
| `'sourceStyle'` | `add-preset` | Optional provenance/style note. Default is `"none"`. |
| `'recommendedUse'` or `'use'` | `add-preset`, `set-use` | Personal use note. Built-ins and user presets default to `"none"` unless you set it. |

## Palette Groups

The GUI uses a short, neutral group list so the palette browser stays easy to scan. User-added palettes may also carry a custom group name; those custom groups remain visible in `ColorKit('catalog')` and searchable by keyword, while the GUI dropdown stays fixed.

| PaletteGroup | Meaning |
|---|---|
| `All` | Show every palette in the catalog. |
| `Built-in` | MATLAB built-in and basic commonly used colormaps. |
| `Sequential` | One-directional continuous variables. |
| `Diverging` | Centered or positive-negative variables. |
| `Categorical` | Categorical data, multiple lines, stations, events, or sample groups. |
| `Cyclic` | Phase, azimuth, orientation, and periodic variables. |
| `Grayscale` | Grayscale or near-monochrome palettes for black-and-white printing checks. |
| `Custom` | Default group for user-defined palettes without a custom group. |

List exact IDs, names, and metadata:

```matlab
catalogTable = ColorKit('catalog');
aboutTable = ColorKit('about');
```

Search examples:

```matlab
ColorKit('search', 'blue')
ColorKit('search', 'diverging')
ColorKit('search', 'Custom')
ColorKit('search', 'MyGroup')
```

Browse before plotting:

```matlab
ColorKit('cards', 'All', 1, 40, 'columns', 4);
ColorKit('cards', 'Sequential', 1, 40, ...
    'nColors', 256, 'columns', 4);
ColorKit('cards', 'blue', 1, 40, ...
    'nColors', 256, 'columns', 4);
```

The card title format is `ID | Name (colors)`. For example, if a card shows `013 | Blue (256)`, both calls below return that palette:

```matlab
C1 = ColorKit('palette', 'Blue', 'nColors', 256);
C2 = ColorKit('palette', 13, 'nColors', 256);
```

## Choosing Palette Types

Sequential palettes are for one-way continuous variables: amplitude, energy, density, probability, intensity, RMS, PSD, or other nonnegative fields.

Diverging palettes are for centered variables: positive-negative fields, differences, residuals, anomalies, perturbations, correlations around zero, or balanced contrasts.

Cyclic palettes are for wrapped variables: phase, angle, azimuth, orientation, direction, or periodic quantities where the first and last colors should connect.

Qualitative palettes are for categories: groups, classes, stations, events, line series, annotations, or labels.

Avoid using `jet` or rainbow-like palettes as the default for quantitative continuous data. They can create false visual boundaries and can be hard to interpret in grayscale. They remain in the catalog mainly for MATLAB compatibility and comparison.

## Recommended Plotting Workflow

Use palette names as variables at the top of a script. This keeps titles, colormap calls, and exports synchronized:

```matlab
mapSurface = "Viridis";
mapLine = "Line8";
mapDiverging = "BlueWhiteRed";
nContinuous = 256;

C_surface = ColorKit('palette', mapSurface, 'nColors', nContinuous);
surf(peaks(120), 'EdgeColor', 'none');
colormap(C_surface);
colorbar;
title(sprintf("surf: %s, %d colors", mapSurface, nContinuous), ...
    'Interpreter', 'none');
```

`Demo_ColorKitFull.m` uses this pattern.

## Image Color Picking

Use `pick` when you want to build a palette from specific pixels in a photo, figure, map, or design reference.

Interactive use:

```matlab
[fileName, folderName] = uigetfile( ...
    {'*.png;*.jpg;*.jpeg;*.tif;*.tiff;*.bmp', 'Image files'; '*.*', 'All files'}, ...
    'Import an image for color picking');

img = imread(fullfile(folderName, fileName));
[C, info] = ColorKit('pick', img, 'preview', true);
```

Click several colors, then press Enter. The output `C` is an `M x 3` RGB matrix. The second output `info` includes the clicked pixel coordinates in `info.Points`, which is useful for documenting or reproducing the workflow.

Reproducible use:

```matlab
points = info.Points;
C = ColorKit('pick', img, 'points', points, 'preview', true);
```

To turn picked colors into a smooth ramp:

```matlab
C256 = ColorKit('pick', img, ...
    'points', points, ...
    'nColors', 256, ...
    'preview', true);
```

Save the result as a user preset after you like it:

```matlab
row = ColorKit('add-preset', C256, ...
    'presetName', 'My_ImageRamp_1', ...
    'paletteGroup', 'Custom', ...
    'type', 'none', ...
    'sourceStyle', 'none');
```

## Image Theme Extraction

Theme extraction summarizes the dominant colors in an image. It is useful for building qualitative palettes, poster styles, presentation themes, or figure-matched color schemes.

```matlab
img = imread(fullfile('Figures', 'fig1_color_pick_source.png'));

C_grid = ColorKit('theme-grid', 8, img, 'preview', true);
C_cluster = ColorKit('theme-cluster', 8, img, 'preview', true);
```

`theme-grid` quantizes RGB values and ranks the most frequent bins. It is fast, deterministic, and reproducible. It can over-emphasize large backgrounds, white margins, black text, or repeated annotation colors.

`theme-cluster` groups similar colors and returns representative cluster colors. It is usually better for gradients, textured images, and noisy images. It can smooth over small accent colors if they occupy only a tiny image area.

Use `theme-grid` when speed and exact repeatability matter. Use `theme-cluster` when you want a more perceptual summary of a complex image.

## Image Colorbar Extraction

Use `imagebar` when a screenshot already contains a colorbar that you want to reproduce.

Interactive use:

```matlab
[fileName, folderName] = uigetfile( ...
    {'*.png;*.jpg;*.jpeg;*.tif;*.tiff;*.bmp', 'Image files'; '*.*', 'All files'}, ...
    'Import an image that contains a colorbar');

img = imread(fullfile(folderName, fileName));
C = ColorKit('imagebar', img, ...
    'samples', 16, ...
    'nColors', 256, ...
    'preview', true);
```

Click the first and last points inside the colorbar. ColorKit creates equally spaced sample points along the straight line between them.

Reproducible use:

```matlab
endpoints = [x1 y1; x2 y2];
C = ColorKit('imagebar', img, ...
    'samples', 16, ...
    'endpoints', endpoints, ...
    'nColors', 256, ...
    'preview', true);
```

For the included colorbar example:

```matlab
img = imread(fullfile('Figures', 'fig2_colorbar_source.png'));
endpoints = [1756 56; 1756 390];

C_anchor = ColorKit('imagebar', img, ...
    'samples', 16, ...
    'endpoints', endpoints, ...
    'preview', true);

C256 = ColorKit('imagebar', img, ...
    'samples', 16, ...
    'endpoints', endpoints, ...
    'nColors', 256, ...
    'preview', true);
```

`samples = 16` means sixteen anchor colors are sampled at equal intervals between the two endpoints. Increase it to 32 or more for segmented, rainbow-like, or rapidly changing colorbars. For a smooth colorbar, 10-16 anchors is often enough.

## Interpolation Algorithm

For image picking and image colorbar extraction:

1. Anchor colors are sampled from image pixels.
2. Colorbar sample coordinates are generated by `linspace` between the two clicked endpoints.
3. Pixel colors are read with bilinear interpolation, so coordinates do not need to fall exactly on integer pixel centers.
4. Anchor colors are resampled to the requested `nColors` using MATLAB `interp1(..., 'pchip')`.
5. Final RGB values are clipped to `[0, 1]`.

`pchip` is a shape-preserving cubic interpolation. It is smoother than linear RGB interpolation and usually avoids the stronger overshoot risk of spline interpolation. This is a practical default for reconstructing a screenshot colorbar, because the screenshot is already encoded in RGB.

Limitation: this interpolation is RGB-space interpolation, not strict perceptual-space interpolation. For designing a new scientific colormap from scratch, start from a compact built-in palette and refine the result in `ColormapDesigner`.

## Add Your Own Preset

`add-preset` is independent from `pick`. Any good-looking `N x 3` RGB matrix can be saved:

```matlab
C = ColorKit('palette', 'BlueCreamRose', 'nColors', 256);

row = ColorKit('add-preset', C, ...
    'presetName', 'My_BurgundyTeal', ...
    'paletteGroup', 'Custom', ...
    'type', 'none', ...
    'sourceStyle', 'none');
```

The new preset is written to `ColorKit_UserPresets.m` and appears at the end of `ColorKit('catalog')`.

Call it later:

```matlab
C = getPresetColormap('My_BurgundyTeal', 256);
```

## Update Recommended Use

All palettes start with `RecommendedUse = "none"`. This lets you decide later what a palette is good for in your own work.

```matlab
row = ColorKit('set-use', 'My_BurgundyTeal', 'frequency dispersion map');

catalog = ColorKit('catalog');
catalog(strcmp(catalog.Name, 'My_BurgundyTeal'), :)
```

The update is stored in `ColorKit_UserMetadata.m`. Reset it with:

```matlab
ColorKit('set-use', 'My_BurgundyTeal', 'none');
```

Catalog columns:

- `ID`: current catalog order. You can call palettes by ID, for example `ColorKit('palette', 13, 'nColors', 256)`.
- `Name`: stable palette name for `ColorKit('palette', name, ...)`.
- `PaletteGroup`: neutral browsing group shown in the GUI and catalog.
- `Type`: palette behavior: `sequential`, `diverging`, `cyclic`, `qualitative`, or `none`.
- `Colors`: number of anchor colors for custom palettes, or native size for MATLAB built-ins.
- `RecommendedUse`: your personal usage note. Defaults to `none`.
- `SourceStyle`: optional style/provenance label.

## ColormapDesigner GUI

Open:

```matlab
ColormapDesigner
```

The GUI supports:

- PaletteGroup filtering and keyword search
- Palette dropdown linked to the palette library
- editable color nodes with `position`, `R`, `G`, `B`, and `Hex`
- adding and deleting color nodes
- automatic node sorting and clipping into valid RGB ranges
- interpolation modes: `linear RGB`, `pchip RGB`, `spline RGB`, and `perceptual-like`
- colormap length selection: 64, 128, 256, 512, 1024
- reverse, gamma, brightness, contrast, saturation, and truncation
- center color locking for diverging maps
- previews for colorbar, scalar field, diverging field, categorical lines, grayscale, and matrix-like data
- export to workspace, MAT, TXT/CSV, MATLAB function, clipboard code, and preview figures
- apply current colormap to existing figures or axes

Parameter notes:

- `Gamma`: redistributes contrast along the colormap. Values above 1 emphasize one end; values below 1 emphasize the other.
- `Brightness`: adds or subtracts overall lightness in RGB space.
- `Contrast`: stretches or compresses distance from middle gray.
- `Saturation`: increases or decreases colorfulness.
- `Truncate start/end`: uses only a subrange of a colormap, then resamples it back to `N` colors.
- `Center color locking`: pulls the center of a diverging colormap toward white or light gray.

## Notes

- User-added presets are stored separately in `ColorKit_UserPresets.m`.
- User metadata overrides are stored in `ColorKit_UserMetadata.m`.
- Built-in palette names use neutral color-property names. User palettes may use any name and any `paletteGroup` metadata.
- `ramp` is not a separate palette library. It is only a readable alias for requesting a named palette at a specified length.
